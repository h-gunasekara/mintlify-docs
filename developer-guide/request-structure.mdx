# Request Structure

This guide explains how to structure your requests to the Catalog API. All requests follow a consistent format to make integration straightforward.

## HTTP Method and Endpoint

All requests to the Catalog API use the POST method and should be sent to:

```
POST https://catalogai.vercel.app/api/products
```

## Headers

Every request must include these headers:

| Header | Value | Description |
| --- | --- | --- |
| `Content-Type` | `application/json` | Indicates that the request body is in JSON format |
| `x-api-key` | `YOUR_VALID_API_KEY` | Your unique API key for authentication |

## Request Body

The request body must be a JSON object containing the following optional fields:

```json
{
  "query": "string",
  "filters": { ... },
  "page": number
}
```

### Request Parameters

<ResponseField name="query" type="string" optional>
  Natural language search string. The API processes this with intelligent features like synonym expansion and implicit filter detection.
  
  **Example:** `"organic cotton shirt"`
</ResponseField>

<ResponseField name="filters" type="object" optional>
  Object containing filter criteria to narrow down results. See the [Filters](/developer-guide/filters) guide for detailed options.
  
  **Example:** `{"price_min": 20, "price_max": 50, "is_available": true}`
</ResponseField>

<ResponseField name="page" type="number" optional default="1">
  The page number (1-indexed) for pagination.
  
  **Example:** `2`
</ResponseField>

## Example Requests

### Basic Search Request

<CodeGroup>

```json Request
{
  "query": "organic cotton shirt"
}
```

</CodeGroup>

### Search with Filters

<CodeGroup>

```json Request
{
  "query": "jacket",
  "filters": {
    "vendor": "TrailBlazer",
    "is_available": true,
    "price_min": 50,
    "price_max": 200
  }
}
```

</CodeGroup>

### Filters-Only Request

<CodeGroup>

```json Request
{
  "filters": {
    "attributes": {
      "color": "Green",
      "fabric": {
        "linen": true
      }
    },
    "is_available": true
  }
}
```

</CodeGroup>

### Paginated Request

<CodeGroup>

```json Request
{
  "query": "shoes",
  "page": 2
}
```

</CodeGroup>

## Request Validation

The API performs validation on your request:

- If `query` is provided, it must be a string
- If `filters` is provided, it must be a valid filter object
- If `page` is provided, it must be a positive integer

If validation fails, you'll receive an appropriate error response with details about the issue.
