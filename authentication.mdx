The Catalog API uses API key authentication. Include your API key in the `x-api-key` header with every request.

## Get your API key

To obtain an API key:

1. [Contact our team](mailto:founders@getcatalog.ai) to request access
2. We'll provision your unique API key
3. Store it securely in your environment variables


## Using your API key

Include the `x-api-key` header in every request:

<CodeGroup>

```bash cURL
curl -X POST https://catalogai.vercel.app/api/search \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{"query": "t-shirt"}'
```

```javascript JavaScript
const response = await fetch('https://catalogai.vercel.app/api/search', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': process.env.CATALOG_API_KEY
  },
  body: JSON.stringify({ query: 't-shirt' })
});
```

```python Python
import os
import requests

response = requests.post(
    'https://catalogai.vercel.app/api/search',
    headers={
        'Content-Type': 'application/json',
        'x-api-key': os.environ['CATALOG_API_KEY']
    },
    json={'query': 't-shirt'}
)
```

</CodeGroup>

## Best practices

### Environment variables

Store your API key in environment variables:

<CodeGroup>

```bash .env
CATALOG_API_KEY=your_api_key_here
```

```javascript Next.js
// .env.local
CATALOG_API_KEY=your_api_key_here

// In your code
const apiKey = process.env.CATALOG_API_KEY;
```

```python Python
# .env
CATALOG_API_KEY=your_api_key_here

# In your code
import os
api_key = os.environ['CATALOG_API_KEY']
```

</CodeGroup>

### Error handling

Handle authentication errors gracefully:

<CodeGroup>

```javascript JavaScript
const searchProducts = async (query) => {
  try {
    const response = await fetch('https://catalogai.vercel.app/api/search', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': process.env.CATALOG_API_KEY
      },
      body: JSON.stringify({ query })
    });

    if (response.status === 401) {
      throw new Error('Invalid API key');
    }

    if (response.status === 403) {
      throw new Error('API key access denied');
    }

    return await response.json();
  } catch (error) {
    console.error('Authentication error:', error.message);
    throw error;
  }
};
```

```python Python
import requests

def search_products(query):
    try:
        response = requests.post(
            'https://catalogai.vercel.app/api/search',
            headers={
                'Content-Type': 'application/json',
                'x-api-key': os.environ['CATALOG_API_KEY']
            },
            json={'query': query}
        )
        
        if response.status_code == 401:
            raise Exception('Invalid API key')
        
        if response.status_code == 403:
            raise Exception('API key access denied')
            
        response.raise_for_status()
        return response.json()
        
    except requests.exceptions.RequestException as e:
        print(f'Authentication error: {e}')
        raise
```

</CodeGroup>

## Status codes

| Status Code | Description |
| ----------- | ----------- |
| `200` | Request successful |
| `400` | Bad request - invalid parameters |
| `401` | Unauthorized - invalid or missing API key |
| `403` | Forbidden - API key access denied |
| `429` | Too many requests - rate limit exceeded |
| `500` | Internal server error |


## Security considerations

- **Never expose API keys** in client-side code
- **Use environment variables** to store API keys
- **Rotate keys regularly** as a security best practice
- **Monitor usage** to detect unauthorized access
- **Use HTTPS** for all API requests (enforced by default)
