# Response Format

This page provides a comprehensive reference for all response formats and fields returned by the Catalog API.

## Success Responses

Successful API requests return responses with HTTP status code `200 OK` and a JSON body. The JSON response always follows this structure:

```json
{
  "products": [ ... ],  // Array of product objects
  "meta": { ... }       // Metadata about the results
}
```

## Products Array

The `products` array contains a list of product objects that match your search and filter criteria. If no products match, this will be an empty array.

### Product Object

Each product in the `products` array has the following structure:

<ResponseField name="id" type="number">
  Unique identifier for the product.
  
  **Example:** `2458`
</ResponseField>

<ResponseField name="title" type="string">
  The product title or name.
  
  **Example:** `"Oversized Logo Hoodie"`
</ResponseField>

<ResponseField name="description" type="string">
  Detailed product description.
  
  **Example:** `"Iconic oversized hoodie with front logo print. Made with 100% organic cotton for superior comfort and sustainability."`
</ResponseField>

<ResponseField name="price" type="number">
  Current product price in the store's currency.
  
  **Example:** `79.99`
</ResponseField>

<ResponseField name="compare_at_price" type="number | null">
  Original price before any discounts, if the product is on sale.
  
  **Example:** `99.99`
</ResponseField>

<ResponseField name="vendor" type="string">
  Name of the product's vendor or brand.
  
  **Example:** `"EcoWear"`
</ResponseField>

<ResponseField name="is_available" type="boolean">
  Whether the product is currently available for purchase.
  
  **Example:** `true`
</ResponseField>

<ResponseField name="attributes" type="object">
  Additional product attributes, which may vary by product type.
  
  **Example:** 
  ```json
  {
    "color": "Navy Blue",
    "fabric": ["Organic Cotton", "Recycled Polyester"],
    "fit": "Oversized",
    "sleeve_length": "Long Sleeve"
  }
  ```
</ResponseField>

<ResponseField name="images" type="array">
  Array of image objects for the product.
  
  **Example:** 
  ```json
  [
    {
      "id": 8645,
      "url": "https://example.com/images/hoodie-navy-front.jpg",
      "alt": "Front view of navy blue hoodie",
      "position": 1
    },
    {
      "id": 8646,
      "url": "https://example.com/images/hoodie-navy-back.jpg",
      "alt": "Back view of navy blue hoodie",
      "position": 2
    }
  ]
  ```
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the product was created.
  
  **Example:** `"2023-02-15T08:23:47Z"`
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of when the product was last updated.
  
  **Example:** `"2023-08-01T14:56:22Z"`
</ResponseField>

### Image Object

Each image in a product's `images` array has the following structure:

<ResponseField name="id" type="number">
  Unique identifier for the image.
  
  **Example:** `8645`
</ResponseField>

<ResponseField name="url" type="string">
  Full URL to the image.
  
  **Example:** `"https://example.com/images/hoodie-navy-front.jpg"`
</ResponseField>

<ResponseField name="alt" type="string">
  Alt text for the image for accessibility.
  
  **Example:** `"Front view of navy blue hoodie"`
</ResponseField>

<ResponseField name="position" type="number">
  Position/order of the image in the product's image gallery.
  
  **Example:** `1`
</ResponseField>

## Metadata Object

The `meta` object contains information about the search results and pagination:

<ResponseField name="total_count" type="number">
  Total number of products matching the query and filters.
  
  **Example:** `128`
</ResponseField>

<ResponseField name="page" type="number">
  Current page number.
  
  **Example:** `2`
</ResponseField>

<ResponseField name="page_size" type="number">
  Number of products per page.
  
  **Example:** `10`
</ResponseField>

<ResponseField name="total_pages" type="number">
  Total number of pages available.
  
  **Example:** `13`
</ResponseField>

<ResponseField name="query" type="object | null">
  Information about the search query processing (if a query was provided).
  
  **Example:** 
  ```json
  {
    "original": "blue cotton hoodie",
    "processed": "blue cotton hoodie",
    "synonyms_used": ["hoodie", "hooded sweatshirt"],
    "implicit_filters": {
      "attributes.color": "blue",
      "attributes.fabric.cotton": true
    }
  }
  ```
</ResponseField>

## Error Responses

When an API request fails, you'll receive an appropriate HTTP status code and a JSON response body with error details.

### Error Response Format

```json
{
  "error": "Error Type",
  "message": "Human-readable error message",
  "details": { ... }  // Optional object with additional error details
}
```

### Common Error Types

<ResponseField name="400 Bad Request">
  The request was malformed or contained invalid parameters.
  
  **Example:**
  ```json
  {
    "error": "Bad Request",
    "message": "Invalid request parameters",
    "details": {
      "filters.price_min": "Must be a non-negative number"
    }
  }
  ```
</ResponseField>

<ResponseField name="401 Unauthorized">
  Authentication failed. The API key was missing or invalid.
  
  **Example:**
  ```json
  {
    "error": "Unauthorized",
    "message": "Missing or invalid API key"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  The requested resource was not found.
  
  **Example:**
  ```json
  {
    "error": "Not Found",
    "message": "The requested endpoint does not exist"
  }
  ```
</ResponseField>

<ResponseField name="429 Too Many Requests">
  Rate limit exceeded.
  
  **Example:**
  ```json
  {
    "error": "Too Many Requests",
    "message": "Rate limit exceeded. Please try again in 60 seconds.",
    "details": {
      "retry_after": 60
    }
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Something went wrong on the server.
  
  **Example:**
  ```json
  {
    "error": "Internal Server Error",
    "message": "Something went wrong on our end. Please try again later."
  }
  ```
</ResponseField>
