# API Endpoints

The Catalog API provides access to product data through a set of RESTful endpoints. This page documents all available endpoints and their usage.

## Products Endpoint

<ParamField query="POST" path="/api/products" />

The products endpoint is the primary endpoint for accessing product data through the Catalog API. It allows you to search and filter products, with support for various query parameters.

### URL

```
https://catalogai.vercel.app/api/products
```

### Authentication

All requests to this endpoint require an API key passed in the `x-api-key` header.

```bash
curl -X POST https://catalogai.vercel.app/api/products \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_VALID_API_KEY" \
  -d '{
        "query": "t-shirt"
      }'
```

<Info>
  See the [Authentication](/getting-started/authentication) guide for more information on obtaining and using API keys.
</Info>

### Request Body

The request body must be a JSON object containing the following optional fields:

<ResponseField name="query" type="string" optional>
  Natural language search string. The API processes this with intelligent features like synonym expansion and implicit filter detection.
  
  **Example:** `"organic cotton shirt"`
</ResponseField>

<ResponseField name="filters" type="object" optional>
  Object containing filter criteria to narrow down results. 
  
  **Example:** `{"price_min": 20, "price_max": 50, "is_available": true}`
  
  See the [Request Parameters](/api-reference/parameters) page for detailed filter options.
</ResponseField>

<ResponseField name="page" type="number" optional default="1">
  The page number (1-indexed) for pagination.
  
  **Example:** `2`
</ResponseField>

### Response Format

On successful requests, the API returns a `200 OK` status with a JSON body containing:

```json
{
  "products": [
    // Array of product objects
  ],
  "meta": {
    "totalItems": 42,
    "totalPages": 5,
    "currentPage": 1,
    "pageSize": 10
  }
}
```

See the [Response Objects](/api-reference/response-objects) page for detailed information about the response structure.

### Error Responses

The API may return the following error responses:

| Status Code | Description |
| --- | --- |
| `400 Bad Request` | Invalid request parameters |
| `401 Unauthorized` | Missing or invalid API key |
| `429 Too Many Requests` | Rate limit exceeded |
| `500 Internal Server Error` | Unexpected server error |

Error responses include a JSON body with details:

```json
{
  "error": "Bad Request",
  "message": "Invalid filter parameters",
  "details": {
    "price_min": "Must be a positive number"
  }
}
```

### Example Requests

<CodeGroup>

```bash cURL
curl -X POST https://catalogai.vercel.app/api/products \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_VALID_API_KEY" \
  -d '{
        "query": "organic cotton shirt",
        "filters": {
          "price_min": 20,
          "price_max": 50,
          "is_available": true
        },
        "page": 1
      }'
```

```javascript JavaScript (fetch)
const response = await fetch('https://catalogai.vercel.app/api/products', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': 'YOUR_VALID_API_KEY'
  },
  body: JSON.stringify({
    query: 'organic cotton shirt',
    filters: {
      price_min: 20,
      price_max: 50,
      is_available: true
    },
    page: 1
  })
});

const data = await response.json();
```

```python Python (requests)
import requests

response = requests.post(
    'https://catalogai.vercel.app/api/products',
    headers={
        'Content-Type': 'application/json',
        'x-api-key': 'YOUR_VALID_API_KEY'
    },
    json={
        'query': 'organic cotton shirt',
        'filters': {
            'price_min': 20,
            'price_max': 50,
            'is_available': True
        },
        'page': 1
    }
)

data = response.json()
```

</CodeGroup>
